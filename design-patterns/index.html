Software Design Patterns Guide for JavaScript

Explore key software design patterns to enhance your JavaScript skills and create robust applications.

1. The Singleton Pattern

This pattern ensures a class has only one instance...


class Singleton {
    constructor() {
        if (!Singleton.instance) {
            Singleton.instance = this;
        }
        return Singleton.instance;
    }
}

2. The Observer Pattern

This pattern allows a subject to notify observers...


class Subject {
    constructor() {
        this.observers = [];
    }
    addObserver(observer) {
        this.observers.push(observer);
    }
}

3. The Factory Pattern

This pattern provides a way to create objects...


class EnemyFactory {
    createEnemy(type) {
        if (type === 'orc') {
            return new Orc();
        }
    }
}

4. The Module Pattern

This pattern allows encapsulation of private and public methods...


const Module = (function() {
    let privateVar = 0;
    return {
        increment: function() { privateVar++; },
        getPrivateVar: function() { return privateVar; }
    };
})();

5. The Adapter Pattern

This pattern allows objects with incompatible interfaces to work together...


class OldSystem {
    request() { return 'Old system'; }
}
class Adapter {
    constructor(oldSystem) { this.oldSystem = oldSystem; }
    request() { return this.oldSystem.request(); }
}

6. The Composite Pattern

This pattern allows you to treat individual objects and composites uniformly...


class Composite {
    constructor() {
        this.children = [];
    }
    add(child) { this.children.push(child); }
}
