Hoisting

Hoisting is a process of declaring variables and functions and moving them to the top of the scope before execution of the code.

1. Hoisting when using var for variable declaration:

console.log(number); // undefined
var number = 10;
console.log(number); // 10

2. Hoisting when using let & const for variable declaration:

Using let or const throws a ReferenceError before initialization:

console.log(number); // ReferenceError
let number = 10;
console.log(number);

Undeclared variables are not hoisted:

console.log(otherNumber); // ReferenceError
let number = 10;
console.log(number);

3. Temporal Dead Zone of let & const variable hoisted:

console.log(number); // ReferenceError
let number = 10;
console.log(number);

4. All undeclared variables are global variables:

function hoist() {
    a = 2;
    var b = 5;
}
hoist();

console.log(a); // 2
console.log(b); // ReferenceError
